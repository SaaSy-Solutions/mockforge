# MockForge Beta Feedback Form

**Purpose:** Collect structured feedback from beta participants
**Last Updated:** 2025-01-27

---

## Feedback Categories

### 1. Bug Reports

**Template:**
```markdown
## Bug Report

**Title:** [Brief description]

**Severity:** [Critical/High/Medium/Low]

**Steps to Reproduce:**
1. [Step 1]
2. [Step 2]
3. [Step 3]

**Expected Behavior:**
[What should happen]

**Actual Behavior:**
[What actually happens]

**Environment:**
- OS: [Windows/macOS/Linux]
- Browser: [Chrome/Firefox/Safari/Desktop App]
- Version: [Version number]
- Date: [Date]

**Screenshots/Logs:**
[Attach if available]

**Additional Context:**
[Any other relevant information]
```

### 2. Feature Requests

**Template:**
```markdown
## Feature Request

**Title:** [Feature name]

**Category:** [Core/UI/Integration/Performance/Other]

**Description:**
[Detailed description of the feature]

**Use Case:**
[Why is this needed? What problem does it solve?]

**Proposed Solution:**
[How should it work?]

**Priority:**
[High/Medium/Low]

**Additional Context:**
[Any mockups, examples, or references]
```

### 4. Performance Feedback

**Template:**
```markdown
## Performance Feedback

**Feature/Page:** [What was slow?]

**Performance Issue:**
[Describe the slowness]

**Expected Performance:**
[What would be acceptable?]

**Environment:**
- Network: [Fast/Slow/Offline]
- Device: [Desktop/Mobile/Tablet]
- Browser: [Browser and version]

**Metrics:**
- Load time: [Seconds]
- Response time: [Seconds]
- Other: [Any relevant metrics]
```

All feedback is:
1. **Acknowledged** within 24 hours
2. **Categorized** and prioritized
3. **Tracked** in issue tracker
4. **Updated** on progress
5. **Closed** with resolution
